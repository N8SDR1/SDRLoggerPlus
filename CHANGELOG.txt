TCILogger — Change Log
======================

v0.50 Beta  (2026-04-11)
------------------------
- New: RBN (Reverse Beacon Network) Band-Opening Alerts — connects to the
  RBN telnet feed and alerts when VHF/UHF bands open near your location.
  Skimmer grid squares are looked up via QRZ/HamQTH and cached. Distance
  from your grid to the skimmer is calculated via Haversine. Configurable
  RBN server/port (default telnet.reversebeacon.net:7000), bands
  (10m, 6m, 2m, 70cm), distance threshold (Miles/KM), per-band cooldown,
  and optional voice announce via browser TTS.
- New: QSO Log search now includes Remarks and Contest/Event fields.
- New: QSO Log Prev/Next pagination — replaces rows per page instead of
  appending. Page size adjusts dynamically based on panel height.
- New: QSO Log panel now has a vertical resize handle (drag bottom edge)
  even when it is the only panel in its column.
- New: 70cm (420-450 MHz) added to JavaScript freqKhzToBand mapping.
- New: Update notification — checks GitHub releases at startup and shows
  a clickable banner badge when a newer version is available. Links
  directly to the release download page. Toggle on/off in Settings →
  Display → "Check for updates on startup."
- New: DX Cluster panel scrolls when spot count exceeds visible area.
- New: Distance Map panel can be resized taller via drag handle.
- New: Toast notifications support custom duration (used for 15-second
  band-opening alerts).

v0.49 Beta  (2026-04-10)
------------------------
- Fix: Installer build — PyInstaller entry point changed from main.py to
  launcher.py. The v0.47 installer bypassed the launcher entirely, so the
  frozen exe ran main.py directly with relative paths. Databases, config,
  and templates were never routed to %APPDATA%\SDRLoggerPlus, the system
  tray icon never appeared, and the app silently failed to start.
- Fix: Launcher now starts flrig_poller and winkeyer_manager threads
  (added in v0.47 but missing from launcher.py).
- Fix: Launcher sets _APP_SETTINGS_FILE and app.static_folder so settings
  persist and static assets load correctly in the installed build.
- Fix: Launcher seeds app_settings.json into AppData on first run.
- Fix: Club Log credential test switched from checklog.php to the correct
  getlotwstate.php endpoint — a safe GET that returns 200 on valid
  credentials without triggering IP blocking.
- Fix: Help guide updated Club Log reference from checklog.php to
  getlotwstate.php.
- Add: CW Decoder — DSP Filter toggle (off by default). Tightens the matched
  filter bandwidth from 0.5× to 0.85× dit duration for conventional rigs
  (IC-9100, FT-991A, etc.) that lack SDR-grade bandwidth filtering. Off by
  default for SDR/Thetis users where Thetis already provides narrow CW filters.
- Add: CW Decoder — Noise Blanker (NB) toggle (off by default). Clamps impulse
  noise (static crashes, key clicks) at 3.5× running average before the envelope
  detector. For conventional rigs without effective built-in NB. SDR users should
  use Thetis NR2/NR3 instead.
- Add: CW Decoder — Farnsworth spacing support. Inter-character and word space
  models are now tracked independently from dit duration. Operators who send fast
  elements with stretched gaps (common in QRS nets and training) are decoded
  correctly instead of producing run-together or chopped text.
- Add: CW Decoder — AFC multi-signal rejection. When two CW signals are within
  the AFC scan range, the AFC now requires 3 consecutive scans confirming a new
  frequency (>40 Hz from current lock) before jumping. Prevents AFC bouncing
  between adjacent signals on a crowded band.
- Add: CW Decoder — QSB/fading recovery. After a deep HF fade (peak decays to
  <2× noise floor), the peak tracker snaps instantly when the signal returns
  instead of lagging ~1 second through normal EMA decay. Eliminates the
  threshold dead zone that caused missed characters after 20m/40m fades.

v0.47 Beta  (2026-04-10)
------------------------
- Add: K1EL WinKeyer hardware CW keyer — full implementation. Connects via
  USB/serial at 1200 baud, 8-N-2 per K1EL specification. Implements Host Open/
  Close protocol, parser sync (4× 0x13), PINCFG register (cmd 0x09) for key
  output port selection (Port 1, Port 2, or Both), Mode Register (cmd 0x0E)
  for keyer mode (Iambic A/B, Ultimatic, Bug), PTT lead/tail timing (cmd 0x04),
  and WPM speed control (cmd 0x02). Background manager thread auto-connects and
  reconnects every 5 seconds when enabled.
- Add: WinKeyer settings panel in Settings — COM port with auto-detected dropdown,
  Enable checkbox, Key Output select, Keyer Mode select, PTT enable + lead/tail,
  default WPM, and Test WinKeyer button that reports firmware version.
- Add: Amber "WinKeyer" badge on CW page when connected. TX path label shows
  "via WinKeyer". CW priority order: TCI → WinKeyer → HamLib.
- Add: /api/serial_ports endpoint — dynamically lists available COM ports on the
  system. Used by the WinKeyer settings dropdown.
- Add: WinKeyer status line in Debug & Diagnostics report (enabled/open/port/version).
- Add: Dedicated WinKeyer section in Quick Start Guide — setup instructions,
  comparison table, COM port sharing notes, and HamLib combo tips.
- Add: WinKeyer + HamLib entries added to Settings Reference table in help guide.
- Fix: PINCFG register was sent to command 0x0E (Mode Register) instead of
  command 0x09 (Pin Configuration). Key output port and PTT were never actually
  configured in hardware — WinKeyer accepted text but had no output routed.
- Fix: serial.TWOSTOPBITS corrected to serial.STOPBITS_TWO (pyserial constant).
- Fix: WinKeyer test endpoint now returns "version" field in JSON response.
- Fix: WinKeyer test button sends Content-Type: application/json and checks
  res.ok before parsing JSON — eliminates "Unexpected token" console errors.
- Fix: Added res.ok guards to all periodic pollers (TCI, flrig, HamLib, digital
  events) to prevent JSON parse errors when an endpoint returns non-JSON.
- Fix: Club Log credential test rewritten — now uses checklog.php (safe read-only
  endpoint) instead of realtime.php. Previous approach sent a test ADIF record to
  the upload endpoint, which counted as a failed upload attempt and triggered
  Club Log's reactive IP firewall on credential errors.
- Fix: Club Log one-strike rule enforced — on any HTTP 403 from realtime.php,
  all further uploads are immediately disabled (_clublog_blocked=True) until the
  user corrects credentials and re-tests. Prevents cascading IP bans.
- Fix: Club Log upload now includes User-Agent header (SDRLoggerPlus/version)
  and logs the upload attempt and response to the debug report.

v0.45 Beta  (2026-04-08)
------------------------
- Remove: OmniRig rig control interface removed entirely. OmniRig COM mode changes were
  silently ignored by the IC-9100 (COM STA cross-process SetSimplexMode limitation).
  Use TCI (Thetis), flrig, or HamLib for rig control. All OmniRig references removed from
  UI, settings, help, and wiki.
- Fix: WinKeyer — serial port now opened as 8N2 (8 data bits, no parity, 2 stop bits) per
  the K1EL WinKeyer hardware specification. Previously used 8N1, causing serial framing
  errors that prevented the device from decoding host commands correctly.
- Fix: WinKeyer — dsrdtr=True now used when opening the port so DTR is asserted. The WKmini
  requires DTR to be active to recognise a host connection; forcing DTR=False (previous
  behavior) silently blocked host mode from engaging.
- Fix: WinKeyer — Mode Register command corrected from 0x01 (which is the Sidetone frequency
  command) to the correct 0x0E (WK2/WK3 Mode Register). Previously, keyer mode and PTT
  settings were written to the wrong register and had no effect.
- Fix: WinKeyer — Host Open sequence now matches the reference implementation: Admin:Close
  is sent first, followed by a 1-second settle delay, then Admin:Open with a 500 ms read
  window. The previous 150 ms window was too short for some firmware versions to respond.
- Fix: WinKeyer — Host Close command corrected from `[0x00, 0x09]` (which is "Get WK1
  Version") to the correct `[0x00, 0x03]` (Admin: Host Close).
- Fix: WinKeyer — flush() called after every write to guarantee bytes are delivered through
  the USB-CDC (CH340) driver to the device immediately on Windows.
- New: WinKeyer settings expanded — Key Output port (PORT1 / PORT2 / Both), Enable PTT,
  and Keyer Mode (Iambic A / Iambic B / Ultimatic / Bug) configurable in Settings.
- Fix: CW page — audio ScriptProcessor buffer size corrected. Previously computed as
  sampleRate × 0.020 = 960 (at 48 kHz), which is not a power-of-two and caused a
  "buffer size must be 0 or a power of two" browser error. Now rounds up to the next
  power-of-two (1024 at 48 kHz).
- Fix: CW page — connection badge now correctly shows "WinKeyer" (amber) when WinKeyer is
  the active CW path. Previously showed "HamLib" if HamLib was also enabled.
- Fix: Club Log — API key field removed from settings. Users only need their Club Log email
  and Application Password (no developer key required).
- New: Club Log — Upload Designator field added. A single letter (default "B") is recorded
  against each uploaded QSO to prevent duplicate uploads on future sessions.
- Fix: HamLib — IC-9100 model number corrected from 368 to 3068 in the Settings page
  example and the help guide rig table.
- Fix: HamLib — Band and mode changes from the QSO entry panel now suppress the HamLib
  frequency/mode poller for 4 seconds, preventing the form from reverting immediately after
  a user-initiated change. Previously, the poller overwrote the new mode/band within the
  first poll cycle.
- Fix: HamLib — Mode-only changes (no frequency) from the QSO form are now accepted by the
  /api/hamlib_tune route. Previously the route returned 400 if freq_mhz was absent.
- Fix: HamLib — Full bidirectional mode name map added for both read (rigctld → form) and
  write (form → rigctld). Modes now correctly translate: USB↔USB, CW↔CW, CWR↔CWL,
  PKTUSB↔DIGU, PKTLSB↔DIGL, RTTYR↔RTTY-R, etc.
- Fix: HamLib — Poller interval reduced from 1500ms to 750ms for more responsive mode
  and frequency updates in the QSO form.
- Fix: QRZ — Subscription link updated; cost removed from UI and documentation (link to
  QRZ subscription page instead).

v0.37 Beta  (2026-04-04)
------------------------
- CW Decoder: fixed "no decodes after running for a while" — noiseFloor was fully
  frozen during marks; if band noise rose while a mark was active, noiseFloor fell
  behind the true noise level, the decoder got stuck in lastLevel=true, and decoding
  stopped until a manual restart; fix: noiseFloor can now fall (but not rise) during
  marks — the one-directional update allows floor to track downward if the signal
  dips, while still preventing mark energy from contaminating the noise estimate
- CW Decoder: fixed "frequency change stops decoding" — toneFreqChanged() only reset
  AFC state, leaving noiseFloor/peakPower/envVal calibrated to the old pitch, and the
  matched-filter ring buffer and IQ oscillator holding stale energy at the old frequency;
  fix: full signal-chain reset on tone slider move — envelope state, matched filter
  ring buffer, IQ accumulators, phase accumulator, and decoder edge state are all
  cleared so the new frequency calibrates cleanly from scratch
- CW Decoder: added upper mark duration guard in processEdge — during signal fade the
  SNR dynamic range compresses, inflating norm for noise bursts and creating implausibly
  long "marks" (several seconds); these were passed to bayesLearnMark(), corrupting the
  learned dit timing; marks longer than 8× the current dit estimate are now discarded
  before classification or learning (8× ≈ 480ms at 20 WPM — safely above any real dah)
- CW Decoder: dit/dah boundary refined from 1.5× to 1.565× — direct extraction from
  CWGET binary decoder section (d64 at offset 0x0107BC and mirror at 0x028390, both at
  the timing decision point); the previous 1.5 came from a generic constant appearing
  elsewhere in the binary; 1.565 is the actual threshold value used in both decoder paths
- CW Decoder: asymmetric speed learning added to bayesLearnMark — CWGET binary uses
  "instant fall / slow rise" (0.33 alpha) for element-length tracking; we mirror this
  by applying full alpha when the observed dit is shorter than current model (operator
  speeding up — track quickly) and half alpha when longer (momentary hesitation — wait
  before slowing the model down); this improves tracking when operators accelerate mid-QSO

v0.36 Beta  (2026-04-04)
------------------------
- CW Decoder: bootstrap dit/dah boundary changed from √3 (1.732×) to 1.5× dit —
  CWGET binary analysis found the constant 1.50 appears 13× as the dit/dah decision
  boundary; 1.5× biases toward calling borderline marks as dahs, which is correct for
  real-world CW where operators commonly send short dahs (~1.5–2T) especially at speed
- CW Decoder: hysteresis widened and made asymmetric — CWGET uses two separate thresholds
  (Treshold=320 ON, CwFilterTreshold=175 OFF, ratio=0.547); symmetric ±4% was too narrow
  and caused chatter on signals hovering near threshold; new HYST_ON=6% (start a mark)
  and HYST_OFF=12% (hold a mark); at default 27% threshold: mark starts at 33%, holds
  until norm drops below 15% — substantially more stable on marginal signals
- CW Decoder: Bayesian variance floor now scales with dit duration (10% CV minimum)
  instead of a fixed 25ms²; at 15 WPM (80ms dit) the old floor was σ=5ms = 6% CV,
  making the Gaussian over-sharp and brittle; new floor scales so slow operators always
  get at least 10% tolerance around their learned timing
- CW Decoder: corrected ENV_DKY comment — was "slightly faster so noise spikes clear
  quickly" (backwards: lower alpha = slower change); corrected to describe fast attack
  (~1.7ms half-life) and slow decay (~8ms half-life) with confirmation from CWGET binary
  (attack=0.297, decay=0.0615 confirmed to match our ENV_ATK=0.30, ENV_DKY=0.06)

v0.35 Beta  (2026-04-04)
------------------------
- CW: Gain slider replaced with Squelch (SNR gate) slider — the Gain control was
  mathematically a no-op: norm=(mfVal−noiseFloor)/(peakPower−noiseFloor) cancels any
  gain factor identically in numerator and denominator, and the SNR gate comparison
  (peakPower ≥ k × noiseFloor) also cancels gain; moving the knob from ×0.01 to ×20
  had zero effect on decode quality, signal bar, or any threshold comparison; the
  gain multiplication is removed from both the TCI and microphone audio paths
- CW: Squelch slider (×1.1–×3.0, default ×1.5) now controls the SNR gate directly —
  the minimum signal-to-noise ratio required before the decoder runs at all; this
  mirrors CWGET's separate SquelchThreshold parameter; low setting (×1.1) decodes
  weak/noisy signals at the cost of more false triggers; high setting (×3.0) requires
  a clean strong signal; ×1.5 preserves prior default behaviour

v0.34 Beta  (2026-04-04)
------------------------
- CW Decoder: fixed critical SNR gate dropout between characters — PEAK_FALL reduced
  from 0.03 to 0.003; at ~1400 IQ windows/sec, the old value gave the peak tracker a
  16ms half-life, collapsing it to near-zero during every 3T inter-character space
  (144ms at 25 WPM), which caused the SNR gate to fail between every character; the new
  value gives a 165ms half-life: peak holds at ~55% across a 25 WPM char-space, drops
  appropriately during word gaps (~24% after 7T), and correctly decays to zero after
  ~1.5s of true silence (7 half-lives)
- CW Decoder: noise floor now only updates during confirmed spaces (lastLevel=false) —
  during a mark, mfVal is at signal level; FLOOR_RISE was slowly pulling the noise
  reference upward mid-element, compressing the norm range and risking threshold dropout
  on long dahs; floor now only learns from silence, which is what it should represent
- CW Decoder: bootstrap dit/dah boundary corrected from 2.0× to √3 (≈1.732×) dit —
  √3 is the geometric mean of 1T (dit) and 3T (dah), i.e. the Bayes-optimal decision
  boundary where misclassification probability is equal on both sides; 2.0× was biased
  toward over-calling dits: marks between 1.73T and 2.0T were incorrectly classified
  as dits during the bootstrap phase before the Gaussian model was trained
