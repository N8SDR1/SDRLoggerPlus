SDRLogger+ — Change Log
=======================

v1.08  (2026-04-17)
------------------------------------------
Map upgrade release: new Distance Map style picker for General / POTA
(Voyager / Positron / Dark Matter), full SAT map projection rewrite for
pixel-perfect station-marker alignment, smarter footprint sizing that
mirrors the S.A.T. controller's own MAP panel, and a satellite-name
label that finally refreshes when you switch birds. Also bundles the
RBN band-opening alert fix that closes a stale-WebSocket race on
Settings save. All v1.07 propagation-forecast and v1.06.1 features
roll forward unchanged.

  ── 🗺 DISTANCE MAP STYLE PICKER (NEW) ────────────────────────────────────
- New: Settings → 📡 Station & Rig → "Distance Map Style" lets you pick
  the General / POTA distance-map basemap from three free, no-API-key
  EPSG:3857 tile sets:
    • **Voyager** (default) — bright, color borders + city/state labels
    • **Positron** — very light, minimalist, maximum readability
    • **Dark Matter** — dark theme to match the SAT map
- Live hot-swap: pick a style, click Save, the tile layer is replaced
  in place — no reload, no marker/line/greyline re-init.
- Dark Matter ships pretty inky out of the box, so when chosen the
  per-pane CSS filter flips to brightness 1.30 / contrast 1.10 — country
  borders and labels read cleanly without losing the dark theme. Voyager
  and Positron keep their existing soft-down filter so they don't
  overpower the rest of the dark UI.
- Distance / bearing / great-circle line / DX & skimmer markers / RBN
  spot placement are all coordinate-based — tile choice is purely
  cosmetic and never touches the math.

  ── 🛰 SAT MAP ACCURACY PASS ──────────────────────────────────────────────
- Fix: SAT map station marker was plotted ~10° north of the user's
  actual grid (Hamilton/OH appearing near the ND/Canada border). Root
  cause: NASA EOSDIS GIBS Blue Marble (EPSG:4326) tiles + Leaflet's
  plate-carrée CRS pixel-math combo mis-aligned the marker overlay vs
  the painted tile imagery. SAT map now renders on standard Web
  Mercator (EPSG:3857) using CartoDB Dark Matter tiles — pixel-perfect
  alignment for the marker, the satellite sub-point and the geodesic
  footprint polygon. Same projection the General/POTA map has always
  used, so behavior is consistent across all three modes.
- Fix: SAT map footprint sized too aggressively. Now prefers the CSN
  controller's own satFootprint value when sane (500–10000 km radius
  range), so our circle matches the controller's MAP panel exactly.
  When the controller doesn't report a footprint, we fall back to the
  geometric formula at the 5° minimum-elevation horizon (instead of
  the theoretical 0°), giving a more realistic "useful pass" coverage
  area ~15 % smaller than the old 0°-horizon math. Always rendered as
  a 96-point geodesic polygon so it warps correctly under Mercator at
  high latitude.
- Fix: SAT map satellite-name label (yellow diamond's permanent
  tooltip) was set once at marker creation and never refreshed — when
  the tracked bird changed (e.g. AO-07 → CATSAT) the diamond kept
  showing the previous name until the page was reloaded, even though
  the banner and log correctly updated. Tooltip content is now
  refreshed on every state push when the satellite name changes.
- Fix: SAT map "my position" was plotted near the equator off west
  Africa for stations everywhere. The CSN S.A.T. firmware reports
  gpslat / gpslon in RADIANS (same convention as satLat / satLon),
  but the backend was passing them through as raw numbers — so an
  Ohio station at 41°N / -83°W (≈ 0.71 rad / -1.45 rad) showed up
  near 0°N / -1.5°E. Backend now applies the same |val| ≤ π
  radians-→-degrees heuristic that's used for the satellite
  sub-point.
- Fix: SAT map "my position" was still off after the radians fix
  because the marker was being overridden by the CSN firmware's GPS
  fields, which are unreliable across passes (sometimes off by
  hundreds of km). Frontend now ALWAYS prefers the user's typed
  grid square from Settings; the S.A.T. GPS is used only as a
  last-resort fallback when no grid is configured. Marker tooltip now
  also displays the source ("Settings grid" vs "S.A.T. GPS") and the
  computed lat/lon for at-a-glance verification.

  ── 🛠 OTHER ──────────────────────────────────────────────────────────────
- Fix: RBN band-opening alerts kept firing for bands the user
  un-checked in Settings. Two issues, both fixed: (1) the WebSocket
  `.close()` is async, so the *old* RBN socket could keep delivering
  spots for several seconds after save with the now-disabled band
  still in its server-side filter — handlers are now detached
  immediately on reconnect; (2) `processRbnSpot()` relied entirely on
  the server-side filter and didn't double-check the spot's band
  against the user's current band-checkboxes — now reads live
  settings on every spot and drops anything not in the allowed set.
- All v1.07 propagation-forecast features (📡 VOACAP chip, /propagation
  popup page, CCIR P.372 noise model, link-budget SNR, 24-hour band
  reliability chart) roll forward unchanged. See v1.07 entry below.


v1.07  (2026-04-17)
------------------------------------------
Propagation-forecast release: new in-app HF propagation predictor with
dedicated popup page, Settings tab, CCIR P.372 noise + link-budget SNR
math, and a 📡 VOACAP chip right next to the callsign in the QSO panel.
All v1.06.1 features (satellite map, per-mode dock layouts, mode-toggle
visibility bump, panadapter spot coloring) are rolled forward unchanged.

  ── 📡 PROPAGATION FORECAST (NEW) ─────────────────────────────────────────
- New: **📡 VOACAP chip** inline in the QSO entry panel's Entity info
  line — click to open a dedicated propagation-forecast popup for the
  current callsign (pulls the grid via QRZ/HamQTH). Chip is
  mode-aware: hidden in 🛰 SAT mode (HF prediction doesn't apply to
  LOS satellite passes) and honors a Settings toggle.
- New: `/propagation` route — self-contained popup page (opens in a
  window like /cw does, sized 1560×1120). Self-labels predictions as
  SDRLogger+'s own simplified forecast and links prominently to
  voacap.com for authoritative runs.
- New: Per-path forecast math (all client-side — cannot affect TCI /
  SAT / cluster / solar pollers):
    • Maidenhead grid → lat/lon, great-circle distance/bearing/midpoint
    • Solar zenith at midpoint (full JD / declination / GMST / hour-angle)
    • foF2 day/night diurnal model scaled from SSN (SFI → SSN Kestrel
      conversion)
    • M-factor (MUF/foF2) that grows with path length, LUF with D-layer
      absorption plus K-index degradation
    • Per-band Gaussian reliability curve around 0.85·MUF with steep MUF
      cutoff and LUF absorption floor
    • K-index global geomagnetic penalty curve (1.0 at K≤2 down to 0.25
      at K≥7)
    • **CCIR Rec P.372** atmospheric / man-made noise model by
      environment (quiet / rural / residential / industrial / city) with
      proper f^-0.27-style frequency scaling in 2.5 kHz SSB bandwidth
    • **Link-budget SNR** estimate in dB per band — EIRP = 30 + 10·log(P)
      + G_tx, per-hop free-space loss, ionospheric absorption scaled
      with hops / freq vs LUF / K, minus CCIR noise floor
    • **Auroral absorption** — paths crossing |lat| > 60° with K≥4 take
      up to ~85 % additional reliability hit
    • **Sporadic E** — summer-hemisphere boost on 24/28/50 MHz for
      short paths (400–2300 km), deterministic-per-UTC-hour so the
      24-hour chart doesn't flicker
    • **Long-path** toggle — flips to 40,075 − d and antipodal midpoint
- New: 📡 Propagation Settings tab — TX power (W), TX / RX antenna
  gain (dBi), noise environment, sporadic-E enable, long-path toggle,
  VOACAP-chip on/off. Changes apply live on next forecast render.
- New: **24-hour band-reliability chart** — UTC columns with the
  current hour highlighted in gold; color-coded cells from "closed"
  to "excellent" per band.
- Large, readable popup: 16px base / 20px title / bumped input +
  label contrast (#b8ccde / #9fb3c4 on dark) so fine print reads
  clearly at arm's length. "← Return to SDRLogger+" button in the
  header closes the popup and re-focuses the main window — no new
  tab / window spawned.

- All v1.06.1 features (satellite map panel, per-mode dock layouts,
  mode toggle visibility bump, panadapter spot coloring, QRZ button
  behavior, spothole all-sources fix, header/SAT cosmetics) roll
  forward unchanged. See v1.06.1 entry below for details.


v1.06.1  (2026-04-17)
------------------------------------------
Hotfix on top of v1.06 — UI fixes folded in, plus major SAT-mode additions
(satellite map panel with live ground track) and per-mode dock layouts.

  ── 🛰 SATELLITE MAP PANEL (NEW, SAT MODE ONLY) ────────────────────────────
- New: Dedicated **🛰 Satellite Map** dock panel that swaps in for the
  Distance Map automatically when you enter SAT mode (and swaps back on
  switch to General / POTA). No manual panel juggling.
- New: Full-globe plate carrée (equirectangular) projection rendered on
  NASA EOSDIS GIBS Blue Marble Next Generation imagery. No API key, free
  natural-earth tileset.
- New: Live satellite sub-point (yellow diamond icon, drop-shadowed),
  footprint coverage circle (yellow, semi-transparent fill), beam-heading
  dashed line from your station to the sub-point, and station marker
  (cyan dot) sourced from either saved grid or live S.A.T. GPS fix.
- New: **Ground track overlay** — ~3 forward orbits + ~90 min trailing
  track, propagated client-side via SGP4 (satellite.js). Cyan dashed
  polyline with UTC timestamp labels at both ends. Rendered in a lower
  Leaflet pane so the footprint, beam line, and sat icon always sit
  cleanly on top. Antimeridian wraps split the polyline automatically.
- New: Backend `/api/sat/tle?sat=<name>` endpoint — pulls Celestrak's
  `amateur.txt` feed, caches it in-memory for 6 hours, fuzzy-matches the
  requested satellite name (e.g. `ISS` → `ISS (ZARYA)`, `RS-44` →
  `RS-44 & BREEZE-KM R/B`). Graceful fallback: if a satellite isn't in
  the feed, the ground track skips and the rest of the map still works.
- New: Backend `/api/sat/status` enriched with sub-satellite point data
  — `sat_lat`, `sat_lon`, `sat_alt_km`, `sat_footprint_km`. Handles the
  CSN S.A.T. firmware's radians-vs-degrees convention (values ≤ ~π are
  auto-converted from radians to degrees) and its diameter-vs-radius
  footprint convention (halved in the frontend).

  ── 🧭 PER-MODE DOCK LAYOUTS (NEW) ─────────────────────────────────────────
- New: Dock panel arrangements (columns, order, collapsed state, heights)
  are now saved **independently for each operating mode** — General,
  POTA, and SAT each have their own layout. Drag the Sat Map to col 2 in
  SAT mode, flip to General, and your General arrangement snaps back
  untouched.
- Migration: Existing single-layout users are transparently upgraded —
  the old `_dock` storage is cloned into all three mode slots on first
  launch so nobody loses their arrangement.
- Note: SAT mode is intentionally NOT auto-restored across page reloads
  (consistent with the "only talk to the S.A.T. while actively in SAT
  mode" design). General / POTA pre-seed on first paint so you boot
  straight into the correct layout.

  ── 🎨 MODE TOGGLE VISIBILITY BUMP ─────────────────────────────────────────
- Cosmetic: The General / POTA / SAT mode toggle in the QSO panel header
  is bigger and more legible — font bumped from 9px to 12px, padding
  from 2px/8px to 4px/12px. It's one of the most-used controls in the
  app; it now reads as a proper target instead of a sidebar afterthought.
- Cosmetic: General mode gets a 📓 notepad icon for visual parity with
  🏕 POTA and 🛰 SAT. Toggle now reads: 📓 General / 🏕 POTA / 🛰 SAT.

  ── 🎨 PANADAPTER SPOT COLORING ────────────────────────────────────────────
- New: Category-based coloring for spots pushed to Thetis via TCI. Choose
  between "Single color" (legacy behavior) or "Color by category" in
  Settings → Display → Panadapter Spot Coloring. Categories:
    • Needed Entity  (default #ff3344 red)   — never worked this DXCC entity
    • New Band       (default #ffd700 gold)  — entity worked, but not on this band
    • New Mode       (default #ff8c00 orange)— entity worked on this band, new mode
    • Standard       (default #00e5ff cyan)  — already worked this band+mode
    • Hot List       (default #ff1744 red)   — call appears on your Hot List
  All five colors are user-configurable via HTML5 color pickers, plus a
  "Reset to defaults" button. Single-color dropdown is aligned with the
  category pickers so all color choices sit inline.
- New: Hot List override — any spot whose callsign is on your Hot List
  is always colored with the Hot color regardless of worked-before state.
- Fix: Canonical cache key (`CALL|BAND|MODE` all uppercase) across both
  the DX cluster panel and the panadapter push paths. Previously the
  panadapter path used mixed casing, causing occasional cache misses and
  wrong colors (often New Mode when the spot should have been New Band).
- Debug: set `window._panColorDebug = true` in DevTools console to log
  per-spot color decisions (key, status, new-band/new-mode flags).
- Note: panadapter lifecycle (how long a spot stays visible on Thetis's
  panadapter) is unchanged — Thetis manages that on its own, matching
  pre-feature behavior. No filtering, auto-clear, or repush logic was
  added.

  ── 🎨 HEADER & SAT MODE COSMETICS ─────────────────────────────────────────
- Cosmetic: Removed redundant "SDR Logbook" label from the header banner
  (the app is already titled SDRLogger+). Header now shows just your
  callsign, with extra spacing before the Lightning / Update banners.
- Cosmetic: In 🛰 SAT mode, the azimuth-only Rotator Control panel is now
  auto-hidden. Satellite tracking uses its own AZ/EL rotator via the
  S.A.T. controller, so the dock panel was redundant (and confusing) in
  satellite mode. Reappears automatically on switch to General or POTA.

  ── 🛠 OTHER FIXES ─────────────────────────────────────────────────────────

- Fix: Spothole "Source Filter" — selecting "All sources" no longer
  silently reverts to "DX Cluster only" on save. Caused by a JavaScript
  `||` falsy-empty-string trap (the "All sources" option has value=""),
  which also made the spothole feed run with the most restrictive filter
  even when the user thought they had picked "all". Users who reported
  "hardly any cluster spots" should now see the full feed once the
  intended source filter is selected.
- New: QRZ Button Behavior toggle — Settings → Lookup → QRZ Lookup now
  has a radio choice for what happens when you click the QRZ button on
  the QSO entry panel:
    • "Show callsign info popup" (default — current v1.06 behavior)
    • "Open the callsign's QRZ.com page in a new browser tab"
  Both modes still auto-fill Name and QTH from QRZ/HamQTH.
- Fix: Band Activity heatmap (header bar) now refreshes on a 60s timer
  so old spots decay out of the trailing 30-minute window even when no
  new spots are arriving. Previously the heatmap only updated when a
  new spot came in, so a quiet band could leave the cells frozen on
  stale counts. Tooltip wording also clarified.
- All v1.06 SAT improvements still apply (see v1.06 entry below).

v1.06  (2026-04-16)
------------------------------------------
SAT performance & polish release. Pre-release for testing alongside v1.05.

  ── 🛰 SAT IMPROVEMENTS ──────────────────────────────────────────────────
- New: Mode-Gated SAT Listeners — SDRLogger+ now only listens to and polls
  the S.A.T. while in 🛰 SAT mode. Switching to General or POTA stops the
  UDP listeners (ports 9932 + 1100), the /track polling, and the /adif
  log poller. Ports are freed for other applications. The S.A.T. badge
  remains hidden outside SAT mode (unchanged from v1.05). This is ideal
  for operators who leave the S.A.T. powered on continuously but only
  want SDRLogger+ to interact with it during active satellite sessions.
- New: /adif Background Poller — a third QSO-import path that polls the
  S.A.T.'s /adif endpoint every 2.5 seconds while in SAT mode. Bypasses
  any firmware-side delay in the UDP push (some QSO LOG TYPE formats
  send the UDP push only after QRZ-lookup / CSN-report completes,
  introducing 5-10s of lag). With this poller, QSOs typically appear in
  SDRLogger+ within ~3 seconds of pressing ADD ENTRY on the S.A.T.,
  regardless of which QSO LOG TYPE is selected.
- New: 🧹 Dedupe Button — appears in the SAT status panel connection bar
  next to ⬇ Fetch S.A.T. Log. Scans all SAT QSOs and removes copies that
  match the same callsign + date + sat name within ±5 minutes. Always
  keeps the earliest copy (which has band/freq/transponder data from
  the original UDP push). Used to clean up any legacy duplicates from
  v1.05 — under v1.06, duplicates can no longer occur.
- New: Smarter SAT Dedup Logic — the dedup key now uses (callsign +
  date + sat_name) within a ±5-minute time window, instead of exact-
  minute time matching. Catches the case where the same QSO arrives via
  the UDP push and the /adif poll with timestamps ~60 seconds apart
  (a S.A.T. firmware behaviour where /adif reports the QSO ~60s after
  the original LOG ENTRY press). No more visible duplicates from
  multi-path arrivals.
- Fix: SAT dedup performance — replaced per-QSO full table scan with
  an in-memory dedup index seeded once at startup (last 7 days only).
  Eliminates the 5-7 second lag between pressing ADD ENTRY on the S.A.T.
  and the QSO appearing in SDRLogger+ that some users saw on databases
  with thousands of QSOs.
- Fix: Launcher missing SAT thread starts — sat_udp_listener,
  sat_adif_listener, and sat_log_poller are now correctly started by
  launcher.py in the installed (PyInstaller) build. Previously these
  threads only ran when launching main.py directly during development,
  meaning the installed v1.05 silently failed to bind UDP ports
  9932 / 1100.
- Doc: Recommended QSO LOG TYPE updated from "ACLog" to "N1MM" — testing
  showed N1MM produces the fastest end-to-end response (typically &lt;3s),
  ACLog is noticeably slower, Log4OM is in between. All formats still
  work — N1MM is just the fastest.
- New: GET /api/sat/qso_count — lightweight endpoint that returns just
  the SAT QSO counter (no S.A.T. HTTP call). Frontend polls this every
  500ms in SAT mode for instant new-QSO detection without slamming
  the S.A.T.'s web server.
- New: POST /api/sat/dedupe — backend endpoint behind the 🧹 Dedupe
  button.

v1.05  (2026-04-16)
------------------------------------------
The biggest single release yet — full CSN Technologies S.A.T. controller
integration, an entirely new Awards Dashboard, and System tab improvements.

  ── 🛰 S.A.T. SATELLITE INTEGRATION (NEW) ─────────────────────────────────
- New: SAT Mode — three-way mode toggle (General / 🏕 POTA / 🛰 SAT) in the
  header bar. SAT mode transforms the entire UI for satellite operating:
  satellite-specific QSO entry form, satellite log columns (Callsign,
  Satellite, Grid, Date, Time, Mode, Up↑/Dn↓ Freq, RST, Remarks), and the
  DX Cluster panel is replaced by the live Satellite Status panel.
- New: Live Satellite Status Panel — polls the S.A.T.'s /track JSON API
  every 2 seconds to display:
    • Satellite name and NORAD catalog number
    • Status: IDLE / TRACKING / AOS — PASS ACTIVE / LOS
    • Real-time AZ / EL with color coding (green when above horizon)
    • Max EL, range (km), AOS Az, LOS Az
    • Live Time-to-LOS countdown (turns red under 60 seconds)
    • Elapsed time since AOS
    • Active transponder name + Doppler-corrected uplink/downlink in MHz
    • Full transponder list (all available for tracked satellite)
    • Pass QSO log (real-time as QSOs are made)
    • GPS lock / satellite count / operator grid square
- New: UDP Event Listener (port 9932) — receives the S.A.T.'s broadcast
  protocol events: BOOT, START TRACK, AOS, LOS, TRANSPONDER, QSO, STOP.
  Maintains a live state machine of the current pass.
- New: Real-Time QSO Push (ADIF-over-UDP, port 1100) — when you log a QSO
  on the S.A.T. device, it is instantly pushed to SDRLogger+ via the
  S.A.T.'s QSO LOG TYPE feature. Compatible with N1MM, DXKeeper, HRD,
  Log4OM, ACLog, and MacLogger formats. QSO appears in SDRLogger+ within
  ~2 seconds with a toast notification and automatic log refresh.
- New: One-Click Log Fetch — "⬇ Fetch S.A.T. Log" button pulls the entire
  QSO log directly from the S.A.T. via its /adif endpoint. Smart duplicate
  detection (callsign + date + time-to-minute) ensures no double entries —
  perfect for importing QSOs logged while SDRLogger+ wasn't running.
- New: SAT QSO Auto-Population — Satellite name, uplink/downlink frequency
  (Doppler-corrected, in MHz), uplink/downlink mode, and main Mode field
  all auto-fill from the S.A.T.'s live tracking data. The Mode dropdown
  label changes to "(auto from S.A.T.)" in SAT mode.
- New: SAT Connection Badge — gold "S.A.T." badge in the header bar, only
  visible in SAT mode. Shows "S.A.T. OFFLINE" → "S.A.T. CONNECTED" →
  "S.A.T. SO-50" (with active satellite name) as the connection comes up.
  Matches the existing TCI / FLRIG / HAMLIB badge pattern.
- New: Dedicated SAT Settings Tab — own "🛰 Satellite" tab in Settings
  with controller IP, UDP broadcast port (9932), QSO Log port (1100),
  setup guide, and the exact Windows Firewall commands needed.
- New: SAT ADIF Export Fields — PROP_MODE=SAT, SAT_NAME, FREQ_RX (downlink)
  added to ADIF export for full LoTW / Club Log satellite DXCC credit.
- New: 13 New Database Columns — prop_mode, sat_name, sat_catno,
  transponder_name, uplink_freq, downlink_freq, uplink_mode, downlink_mode,
  aos_az, los_az, my_grid, my_lat, my_lon. Existing databases auto-migrate.

  ── 🏆 AWARDS DASHBOARD (NEW) ─────────────────────────────────────────────
- New: Complete Awards Tracking System — accessible via /awards from the
  main UI. Four trackers, all powered by the worked-DXCC cache:
    • DXCC — 340 entities tracked. Color-coded cards: gold = completed,
      cyan = worked, dim = not worked. Filter by continent, sort by entity
      name or worked count. Per-band breakdown.
    • WAS (Worked All States) — US 50-state map view + card grid.
      Filter by band/mode. Shows which states need confirmation.
      Built from STATE field in QSO records.
    • WAZ (Worked All Zones) — 40 CQ Zones. Each zone shows its name,
      countries within, and worked status. Filter and sort options.
    • WPX (Worked All Prefixes) — CQ WPX-style prefix extraction
      algorithm. Tracks unique callsign prefixes worked. Sortable by
      prefix or worked date.
- New: Server-side worked-prefix cache (_worked_prefixes) joins the
  existing _worked_entities and _worked_zones caches for fast lookups.
- New: /api/awards/dxcc, /api/awards/was, /api/awards/waz, /api/awards/wpx
  endpoints serve the worked data to the dashboard.

  ── ⚙ SYSTEM TAB IMPROVEMENTS ────────────────────────────────────────────
- Change: Replaced the in-app zip-update mechanism with a cleaner
  "Check for Updates" notification + GitHub release link. The
  manualUpdateCheck() flow polls GitHub and shows a one-click link to
  the latest installer.
- Change: /api/update_check now accepts ?force=1 to bypass the cache for
  manual update checks.
- Removed: Dead applyUpdate() JavaScript function and its UI.
- New: Settings Export to backup folder — exportSettings() now saves the
  current settings JSON to the configured Backup Path with a timestamp,
  giving you a portable snapshot before making major changes.
- New: /api/export_settings_to_folder backend endpoint.

  ── 🛰 SAT-RELATED FIXES ──────────────────────────────────────────────────
- Fix: SAT settings modal no longer closes immediately when configuring
  SAT IP / port — onchange="saveSettings()" handlers removed from SAT
  inputs (settings now save when modal closes normally).

v1.03  (2026-04-15)
------------------------------------------
- New: Lightning Detection — three-layer lightning alert system with a
  flashing ⚡ bolt in the banner bar. Colour-coded by proximity:
    • Yellow  — distant lightning (> 25 mi), slow 4-second pulse
    • Orange  — moderate distance (10–25 mi) or NOAA warning, 3-second pulse
    • Red     — close lightning (< 10 mi), fast 2-second pulse
  Three data sources (use any combination):
    • Blitzortung.org — real-time global lightning strikes with distance and
      compass direction from your QTH (free, no API key)
    • NOAA / NWS — severe thunderstorm warnings for your area (US only,
      free, alert-only — no distance data)
    • Ambient Weather — personal weather station lightning sensor (WS-2000,
      WS-5000, and compatible models) via REST API. Requires both an API
      Key and a separate Application Key from ambientweather.net/account/keys
  Configurable range (miles/km) and per-source enable/disable toggles in
  Settings → Cluster & Integrations → Lightning Detection.
  Includes liability disclaimer and acceptance checkbox — users must
  acknowledge the convenience-only nature of the feature before enabling.
  Hover over the ⚡ banner to see last update time and any active NOAA
  warning text. Backend polls every 90 seconds.
- New: Browse Buttons — native file/directory picker dialogs for TQSL path,
  ADIF monitor file paths, and database backup folder settings fields.
- New: ADIF Import Confirmation — the ADIF File Monitor now shows a
  confirmation dialog ("Import X QSOs?") before writing detected QSOs to
  the database. Dismiss advances the file offset without importing.
- New: MSHV Field Stripping — ADIF imports now whitelist only recognised QSO
  fields, automatically discarding extra fields from apps like MSHV
  (STATION_CALLSIGN, MY_GRIDSQUARE, DISTANCE, QSO_DATE_OFF, TIME_OFF).
  ADIF headers (<EOH>) are also stripped before parsing.
- Change: Help icon in banner changed from 📖 book to ? for cleaner appearance.
- Fix: POTA spot sorting — clickable sort headers (Activator, Mode, Freq)
  now work in POTA spots view, matching DX Cluster behaviour.
- Fix: Stroke callsign map/distance — callsigns containing "/" (e.g.

v0.52 Beta  (2026-04-12)
------------------------
- Fix: Update notification now detects pre-releases — the /releases/latest
  GitHub API endpoint skips pre-releases, returning 404 for beta-only repos.
  Falls back to /releases list and finds the most recent non-draft release.
- Major: CW Decoder — comprehensive signal chain overhaul for spacing accuracy
  and decode reliability. Multiple interacting root causes identified and fixed:

  Edge detection: replaced debounce filter (which reset on every threshold
  flicker and swallowed inter-element spaces) with a one-edge lookback buffer.
  The buffer examines completed segments: noise-mark blips (<20% of dit) and
  micro-space flickers (<8% of dit) are merged into surrounding edges. Valid
  edges are never lost.

  Matched filter: reduced from 0.5× to 0.3× dit (DSP mode: 0.85→0.55×).
  The 0.5× window caused ~18 ms of systematic space compression at 20 WPM,
  pushing inter-character spaces toward the inter-element classifier boundary.

  Envelope decay: increased ENV_DKY from 0.06 to 0.09 (half-life 8→5.5 ms)
  for faster mark→space transitions and cleaner space detection.

  Noise floor holdoff: after each mark→space transition the noise floor is
  frozen (no upward tracking) for 1.2× dit. This prevents the envelope decay
  tail from contaminating the floor and compressing normalization for the first
  element of the next character.

  Peak re-snap: after long spaces (>5× dit), peakPower is snapped instantly
  to the new signal level instead of relying on slow EMA tracking. Ensures
  correct normalization at word boundaries.

  Space classifier: widened Gaussian variances (elem 2→3×, char 4→6×,
  word 8→10× _bDitVar) for smoother crossover between distributions.
  Separated element/character space clamp ranges (elem max 1.2T, char min 2.0T)
  to prevent model collapse.

  Bootstrap: reduced BAYES_MIN from 8 to 5 marks so the learned Bayesian
  model activates sooner — fixed ratio thresholds are less accurate.

  Sample-based timing: replaced performance.now() with monotonic sample
  counting (_sampleTimeMs). Eliminates 1–5 ms JS event-loop jitter that
  added noise to edge duration measurements.

- Improve: CW Decoder — Default threshold raised from 27% to 35%.
- Improve: CW Decoder — Threshold slider range extended to 15–90%.
- Improve: CW Decoder — Proportional hysteresis replaces fixed constants.

v0.34 Beta  (2026-04-04)
