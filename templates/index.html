TCI Interface — Expert Electronics Thetis / EESDR direct SDR control

Connects to Thetis or EESDR via WebSocket TCI for real-time frequency, mode, callsign decode, and CW keying. Enable only if running Thetis or EESDR — disable if using flrig or HamLib only. TCI is first-priority rig path when enabled.
Status: —

ITU Region — band plan · out-of-band VFO alerts

Sets the ITU band plan used for band auto-detection and out-of-band VFO warnings. Region 1 = Europe / Africa / Middle East / Russia  ·  Region 2 = Americas (default)  ·  Region 3 = Asia-Pacific

flrig — W1HKJ XML-RPC rig control · all major brands · CW keying

Connect any flrig-supported rig for frequency/mode following, DX spot tuning, and CW keying. Run flrig, select your rig and COM port in its window, then enable here. flrig takes priority after TCI for rig control.
Digital: audio-passthrough modes (FT8, FT4, JS8, WSPR, PSK31). Use USB-D for Icom · DATA-U for Kenwood/Yaesu · blank to auto-detect.  |  RTTY: leave blank for native FSK (default). Set to USB-D if using fldigi or other AFSK software.
Status: —  ·  Download flrig ↗  ·  Help ↗

HamLib / rigctld — Yaesu · Icom · Kenwood · 300+ rigs

SDRLogger+ connects to rigctld (the HamLib daemon) via TCP — it does not talk to the radio directly. You must start rigctld.exe separately first with your rig model and COM port, then enable HamLib here.
Example: rigctld.exe -m 3068 -r COM3 -s 19200  (IC-9100 = model 3068 · find yours at the HamLib rig list ↗)
SDRLogger+ then connects to rigctld on the host/port below. TCI and HamLib can run simultaneously. ⚠ rigctld and flrig cannot share the same COM port — disable flrig before starting rigctld on that port.
Status: —  ·  HamLib ↗  ·  Supported rigs ↗

K1EL WinKeyer — Hardware CW Keyer

Connect a K1EL WinKeyer (or compatible clone) via USB/serial for hardware CW keying. SDRLogger+ speaks the WinKeyer host protocol at 1200 baud (8-N-2).
CW priority: TCI → WinKeyer → HamLib. Enable WinKeyer here if you don't have TCI and want dedicated CW keying hardware.
Status: —  ·  K1EL WinKeyer ↗

LoTW (ARRL Logbook of the World)

Uploads each QSO to ARRL LoTW automatically on save.
Requires TQSL (TrustedQSL) to be installed and your LoTW callsign certificate already loaded in TQSL.
Default install path: C:\Program Files (x86)\TrustedQSL\bin\tqsl.exe
Station Location is the named location you set up in TQSL (File → Station Locations). Using a Station Location is more precise than callsign alone and is required if you operate from multiple locations or hold multiple callsigns. If you enter just tqsl as the path, TQSL must be on your Windows System PATH (System → Advanced → Environment Variables → Path).

Lightning Detection — ⚡ banner alert when lightning is detected near your QTH

Polls lightning data sources and shows a flashing ⚡ alert in the banner when strikes are detected within your configured range. Useful for HF noise awareness and antenna safety.
⚠ DISCLAIMER
Weather alerts (lightning and high wind) are provided as informational awareness only. They are approximations based on public data sources (NWS/NOAA, Blitzortung, Ambient Weather) and may be delayed, incomplete, or inaccurate. Do NOT rely on SDRLogger+ as your primary safety system. Always use official weather services and your own judgment. By enabling weather alerts you acknowledge this limitation and accept responsibility for your own station and antenna safety.
Data Sources
AMBIENT WEATHER API KEYS — from ambientweather.net/account/keys
Two keys are required: your API Key (created under your account) and a separate Application Key (click "Developers: Click here to create one" on the same page and provide an app description). They are different keys.
ECOWITT API CREDENTIALS — from ecowitt.net account → API Setting (the gateway MAC is on the same page or in the WS View Plus app under Devices)
Three values are required: Application Key, API Key, and your gateway's MAC address (e.g. 34:94:54:AB:CD:EF). Same credentials power both Lightning and the Wind source below.

High Wind Alerts — 💨 banner alert when high wind is detected at your QTH

Polls NOAA/NWS active alerts, your nearest METAR station, and/or your Ambient Weather PWS for sustained and gust wind data. Shows a 💨 banner at three severity tiers (yellow/orange/red) so you can make the call on parking antennas or disconnecting coax before a front arrives.
Data Sources
Look up your nearest station at aviationweather.gov/data/metar or metar-taf.com — any airport ATIS / ASOS within ~30 miles works.
Banner tiers: elevated (yellow) ~10 mph below threshold · high (orange) at/above threshold · extreme (red, bouncing) on NWS High Wind / Extreme Wind Warning or 15+ mph above threshold.

🛰 S.A.T. Controller — CSN Technologies satellite controller by K. Bush AC2MI

Switch to SAT mode using the GENERAL / 🛰 SAT toggle in the header. Satellite contacts are stored in hamlog.db with PROP_MODE=SAT.
The S.A.T. broadcasts live tracking data (AZ/EL, Doppler, transponders) and can send QSOs via its QSO LOG TYPE feature.
S.A.T. event broadcasts (START TRACK, AOS, LOS, etc.)
Receives ADIF records from S.A.T. QSO LOG TYPE

S.A.T. QSO Log Setup — configure on the S.A.T. device under LOCATION settings

On the S.A.T. web UI → LOCATION section, configure:
  • QSO LOG TYPE: N1MM ⭐ recommended (fastest QSO push). Other formats also work — ACLog is noticeably slower, Log4OM is in between.
  • QSO LOG IP: Your PC's IP address
  • QSO LOG PORT: Must match the "QSO Log Port" above (default 1100)

When you log a QSO on the S.A.T. (and you're in 🛰 SAT mode in SDRLogger+), the QSO appears in the SAT log within ~3 seconds with satellite, transponder, and Doppler data.

Note: SDRLogger+ only listens to the S.A.T. while you're in 🛰 SAT mode. Switching to General or POTA stops the listeners and frees the UDP ports.

Windows Firewall — required for UDP reception

If not receiving data, open an Administrator command prompt and run:
netsh advfirewall firewall add rule name="SDRLogger+ SAT UDP" dir=in action=allow protocol=UDP localport=9932
netsh advfirewall firewall add rule name="SDRLogger+ SAT ADIF" dir=in action=allow protocol=UDP localport=1100
